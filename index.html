Today, none of these can do this

  • Manus — hands you a payment link after 20 minutes
  • ChatGPT Agent — redirects to merchant's app for payment
  • Claude Connectors — search only. "Book on booking.com" → deep link out
  • OpenAI Operator — drives human GUIs. No registration, no payment
  • Gemini — navigates app, then hands back for final payment tap
  • Alexa+ / Siri — zero commerce endpoints

Why not a simpler approach?

  • Why not MCP? MCP assumes stateful sessions between agent and tool. Commerce is stateless — every purchase is a new transaction with its own mandates. REST endpoints are the right primitive. ADR-0006: REST over MCP →
  • Why not an SDK? An SDK that handles RSA keys and payment mandates is a supply-chain attack vector. A markdown spec is inert, versioned by kiosk.tech, and reviewable by every merchant before deployment. Agents generate their own integration. ADR-0003: Spec over SDK →

getgroceries.com runs Kiosk — an open protocol that makes any service reachable by AI agents. Your store gets a /.well-known/kiosk.json endpoint. Agents discover, register, browse, order, and pay — all via REST. No intermediary. No transaction fee. Apache 2.0.
